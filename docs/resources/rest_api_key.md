# pfsense_rest_api_key (Resource)

Creates a pfSense REST API key via the REST API v2. The key value is only available at creation time and stored in Terraform state. Import is not supported.

~> **Note:** The `key` attribute is only populated when the resource is first created. Subsequent reads will preserve the key from state but cannot recover it from pfSense. If the state is lost, the key must be recreated.

## Example Usage

```terraform
resource "pfsense_rest_api_key" "terraform" {
  description  = "Terraform automation"
  hash_algo    = "sha256"
  length_bytes = 24
}

resource "pfsense_rest_api_key" "monitoring" {
  description = "Holmes monitoring agent"
}
```

## Schema

### Optional

- `description` (String) Description for the API key (max 128 characters). Changing this forces a new resource.
- `hash_algo` (String) Hash algorithm for the key. One of `sha256`, `sha384`, `sha512`. Default: `sha256`. Changing this forces a new resource.
- `length_bytes` (Number) Key length in bytes. One of `16`, `24`, `32`, `64`. Default: `24`. Changing this forces a new resource.

### Read-Only

- `hash` (String, Sensitive) The hash of the API key as stored by pfSense.
- `id` (Number) The numeric ID of the API key.
- `key` (String, Sensitive) The API key value. Only available at creation time.
- `username` (String) The pfSense user who owns the key.
